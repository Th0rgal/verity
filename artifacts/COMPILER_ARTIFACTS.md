# Compiler Artifacts

This folder collects compiler artifacts and fixtures used by tests, CI, and documentation.

## Contents

- `yul/`: Generated Yul output used by Foundry tests and differential testing. Generated by `lake exe verity-compiler`.
- `macro_property_tests/`: Auto-generated Foundry property test stubs for all `verity_contract` contracts. Generated by `scripts/generate_macro_property_tests.py`.
- `verification_status.json`: Machine-readable verification metrics (theorem counts, axiom counts). Generated by `scripts/generate_verification_status.py`.
- `layer2_boundary_catalog.json`: Machine-readable Layer 2 theorem target and proof-boundary catalog. Generated by `scripts/generate_layer2_boundary_catalog.py`.
- `lean_warning_baseline.json`: Lean warning regression baseline. Consumed by `scripts/check_lean_warning_regression.py`.
- `interpreter_feature_matrix.json`: Machine-readable interpreter feature support matrix.
- `evmyullean_capability_report.json`: EVMYulLean builtin coverage report. Generated by `scripts/generate_evmyullean_capability_report.py`.
- `evmyullean_adapter_report.json`: EVMYulLean adapter function report.
- `evmyullean_unsupported_nodes.json`: EVMYulLean unsupported AST node report.
- `issue_1060_progress.json`: Issue #1060 migration tracking ledger (complete).

## Related Source

- Compiler implementation: `Compiler/`
- Proofs: `Compiler/Proofs/`
- Tests: `test/` (Foundry unit, property, and differential tests)

## Assumptions

Yul is compiled to bytecode using `solc` (trusted), and the runtime semantics are modeled in Lean. See:

- Yul semantics: `Compiler/Proofs/YulGeneration/Semantics.lean`
- Codegen correctness: `Compiler/Proofs/YulGeneration/Preservation.lean`
